# 关键风险与技术决策

## Fn 按住监听风险（高）
- 风险：Fn 在 macOS 上常被系统当作修饰键处理，不一定稳定产生可捕获的按下/松开事件。
- 决策：
  - 必须支持 Fn：实现上采用 `CGEventTap` 尝试捕获。
  - 同时允许用户配置其他快捷键作为替代方案。
  - UI 可提示：若检测到 Fn 捕获不可用，引导用户切换自定义快捷键。

## 全应用文本注入（高）
- 风险：不同应用/控件对 AX 写入支持不一致，且需要辅助功能权限。
- 决策：
  - 首选 AX 注入；失败降级剪贴板保证可用。
  - 对无法注入的场景统一走降级，不做特判堆逻辑。

## STT 方案与路由（中-高）
- Whisper API：效果好但依赖网络与成本，且可能不天然支持真正 streaming。
- Apple Speech：交互好但效果/语言/权限受限。
- 决策：实现 `Transcriber` + `STTRouter`，支持 Whisper / Apple Speech / Custom。

## 音频格式（中）
- PRD 建议 MP3，但原生 MP3 编码依赖第三方或额外工作。
- 决策：v1 建议先用系统友好的 `m4a(AAC)` 存储与播放；后续可按需要增加 MP3 编码。

## LLM Streaming 解析（中）
- 风险：不同 OpenAI-compatible 服务对 SSE 格式实现细节差异。
- 决策：
  - 实现稳健的 SSE 解析器
  - 对异常分片与非标准行为容错
  - 失败自动重试最多 3 次

## 编辑模式 prompt 约束（中）
- 风险：LLM 输出夹带解释文本导致替换结果污染。
- 决策：prompt 明确要求只输出最终文本；客户端额外做轻度清洗（例如去除 ``` 包裹）作为保护。
