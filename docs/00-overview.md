# 项目概览

## 背景
在 macOS 上实现一款“按住说话、松开即输入”的语音输入法工具：常驻菜单栏，通过快捷键触发录音并将语音转文字后直接插入当前光标位置，同时同步到剪贴板；提供可配置的 LLM 服务接入（OpenAI 兼容协议）、历史记录管理与导出能力，并支持“选中文本后语音指令编辑替换”的增强功能。

## 目标（从 PRD 提炼）
- 按住触发键开始录音；松开立即结束录音并进入处理流程。
- 两种模式：
  - 普通输入模式：语音转文字后插入光标。
  - 文本编辑模式：选中文本后触发，把“选区文本 + 语音指令转写文本”交给 LLM，输出纯替换文本并替换选区。
- 管理界面：
  - 快捷键配置立即生效。
  - LLM 配置（OpenAI-compatible）。
  - STT 配置（Whisper/OpenAI-compatible transcriptions + Apple Speech 作为备选/降级）。
- 历史记录：近一周、包含音频文件与最终文本，支持播放/导出（Markdown）/清空。
- 兼容性强约束：文本注入尽量覆盖所有应用；不行则降级为剪贴板方案。

## 关键交互
- 菜单栏常驻，展示状态（就绪/录音中/处理中/失败）。
- 录音浮层固定显示在屏幕正中央下方，展示：状态、波形、计时。

## 明确的产品决策（来自用户确认）
- 必须支持 Fn：默认触发键为 Fn（按住触发，松开结束），用户可设置其他快捷键。
- STT：默认使用“Whisper/OpenAI-compatible transcriptions”（本文后文称 `WhisperProvider`），同时支持 Apple Speech（`AppleSpeechProvider`）作为备选。
- 文本编辑模式：语音指令先转文字，再与选区文本一起喂给 LLM；LLM 输出为纯替换后的文本（无解释）。
- 历史记录导出：Markdown。

## 非目标
- 本期不做历史加密。
- 不做账号体系与跨设备同步。
